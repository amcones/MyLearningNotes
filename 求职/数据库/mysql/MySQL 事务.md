## 事务四大特性

- 原子性
- 一致性
- 隔离性
- 持久性

### 事务的实现

- 原子性：undo log（回滚日志）
- 持久性：redo log（重做日志）
- 隔离性：MVCC（多版本并发控制）或锁
- 一致性：持久性+原子性+隔离性

## 并发事务问题

- 脏读：读到其它事务还未提交的数据
- 不可重复度：前后两次读数据不一致
- 幻读：前后读到的记录数量不一致

## 事务隔离级别

- 读未提交（能读到其它事务未提交的数据）
- 读已提交（只能读到其它事务已经提交的数据）
- 可重复读（事务执行中多次读取结果相同）
- 串行化（单线程）

### 隔离级别实现

- 读未提交直接读最新数据
- 串行化加读写锁
- 读已提交/可重复读 使用 Read View 实现，前者是每句执行前生成一个 Read View，后者是事务开启时生成一个 Read View，整个事务期间都使用。
